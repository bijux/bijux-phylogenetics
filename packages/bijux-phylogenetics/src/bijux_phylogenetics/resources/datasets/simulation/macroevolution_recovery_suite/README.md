# Macroevolution Recovery Suite

This packaged suite groups the governed macroevolution recovery surfaces that
already exist separately in the public runtime:

- `continuous_mode_recovery_panel`: Brownian, OU, early-burst, and transformed
  continuous-trait recovery against stored local `geiger` fits
- `discrete_mode_recovery_panel`: ER, SYM, and governed ARD discrete-trait
  recovery against stored local `geiger` fits
- `known_answer_reference_panel`: stored-truth topology, parameter, node-state,
  and branch-event recovery checks

The suite exists because these three surfaces answer one larger question that
the individual panels do not answer alone: whether Bijux owns one coherent
macroevolution recovery story across continuous models, discrete models, and
truth-anchored simulation review.

## Governed Outputs

The `expected/` directory stores the suite-level workflow bundle generated by
the public runtime surface. It includes:

- `workflow-summary.tsv`: suite-level aggregate counts across governed recovery
  evaluations
- `component-summary.tsv`: one row per component panel with its bundled review
  counts
- `requirement-summary.tsv`: explicit pass or fail rows for the suite-level
  contract
- `sim-char-summary.tsv`: the governed `geiger::sim.char` parity envelope
- `components/`: bundled outputs from the governed continuous, discrete, and
  known-answer recovery panels

## Public Surface

The package-owned public surface is
`run_macroevolution_recovery_suite_demo(...)`. It materializes the component
datasets, reruns the governed component recovery workflows, writes one live
suite bundle, and produces one overview that ties the three recovery
components together.
