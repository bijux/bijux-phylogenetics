# Reports

This directory holds durable example reports and reviewer-facing proof outputs
owned by the repository.

Generated reports created during local verification should go to `artifacts/`
unless the task explicitly requires a tracked report surface in this tree.

Level 1 trust reporting now includes workflow-validation and release-gate
surfaces that summarize checked-in fixture coverage, known failure behavior,
workflow maturity, and retained-versus-excluded taxon traceability.
