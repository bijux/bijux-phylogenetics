# Runtime System Map

The runtime is organized around a few durable capability families:

- trees: validation, inspection, comparison, rooting, rendering
- alignments: trimming, coding diagnostics, translation, identity summaries
- comparative analysis: signal, Brownian/OU fits, PGLS, multiple testing
- ancestral reconstruction: continuous and discrete state workflows
- adapters: governed orchestration of external alignment and inference tools
- outputs: bundles, reports, figures, and review artifacts

This is the map public readers need first. It groups behavior by domain
responsibility instead of by internal module history.
