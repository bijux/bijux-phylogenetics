# Package Map

The repository currently publishes two public install surfaces and one
maintainer surface.

## Published Runtime Surfaces

- `bijux-phylogenetics`
  Canonical runtime package and documentation owner for the product surface.
- `phylogenetic`
  Compatibility alias package for the same runtime family.

## Maintainer Surface

- `bijux-phylogenetics-dev`
  Repository-only tooling for documentation, governance, release, and evidence
  maintenance.

The runtime package should stay the center of gravity for public behavior.
Alias and maintainer surfaces should not compete with that ownership.
